How to create new Kits for BG2 - Shadows of Amn / Throne of Bhaal

-By Daxziz-

Files to Edit are all the necessary files to create a kit for BG2!
You should start with take a look at them!!!

EXPLANATION OF FILES:

ABCLASRQ.2DA - This is the file which control the minimum stats of your kit, under the creation!

ABDCDSRQ.2DA - This is the file which control the minimum stats requirements you have to have in
order to DUAL-CLASS to your kit. (Since you not are able to dual class to a kit in BG2, then is
pretty much worthless!)

ABDCSCRQ.2DA - This one controls the minimum stats for DUAL CLASSING from our kit into another!

ALIGNMNT.2DA - This is the file which control the alignment of your kit.

CLASWEAP.2DA - This file control which kind of weapons your kit can use.

KITLIST.2DA - Now, This is an important file, this file control allot of in game stuff
such as Class Description, Alias to innate abilities files etc.

KITTABLE.2DA - This file control which races, which can play your kit!

LAYHANDS.2DA - This file control the Lay on Hands abilities! (Tricky)

WEAPPROF.2DA - This file control the masteries that your kit will be allowed.

Note by Manveru : Check 2DAs section for more 2DA files description to edit more sophisticated kits features.

Now when you know a little about the files, can we start the creation of your Kit!

Hold it, do we want to create our very own CLAB files? Sure we do... we are editors after all, right?
If you have chosen to do as I wrote above, then go straight to Step 7! If you want to learn more about CLAB-files,
then continue reading!

The clab-files control the abilities your new kit gets through his level ups.
Try take a look at the monk, he gains a lot of special attacks, as abilities, this is written in the CLAB-files.

Here is an example:

2DA V1.0
****
            1              2               3
ABILITY1    GA_MYTHING     ****            **** 
ABILITY2    ****           GA_MYTHING2     **** 
ABILITY3    ****           ****            GA_MYTHING3


Here can you see an example of how the CLAB-file functions... Ability 1 is gained as level 1, Ability 2 at level 2 etc. 
Of course can they be mixed, but you'll figure out later on, it's not difficult, but you'll find out soon, as I said.

MYTHINGx refers to spells, made as Innate ( abilities ) in the Spell Maker ( found in IEEP ). BUT they refers to the NAMES 
of the files ( in windows ), which shall be in the override folder.

Now am I sure that you wonder why it says GA_ in front of the filenames, this is also a thing you'll have to know, for being able to understand it. And I guess that you'll be even more confused when I say, that you also can write AP_ in front of the original filename. But here is the key: GA means Gain Ability, gives much more sense now, but if you still are doubling why it's there, then imagine that it's a command, saying that when you becomes Level 2 then do you get ability 6, ( look it DOES NOT have to be the same level and ability ). Well GA is a ability which you'll be the one to decide when to use, but AP is
quite different, it activates the ability, and you'll not be able to decide when you want to use it or not. So this is why, AP only should be used in cases where you get a item ( yes a item, you are able to give your kit a new item, when it becomes level x ). But it can of course be used in other cases, but you'll find out, when you'll need it.

So this is actually how a CLAB file functions, it's pretty easy when first you have found the logic, so here I recommend you to remember the differences between GA_ and AP_, and do also always remember the _ (underscore) after GA and AP, it won't work otherwise. Do also remember that the item or spells name ( in widows ), not may exceed 8 letters. Do also remember to extract the spell you want to use, and modify it in Spell Maker to a ability, forgetting this, will just result in a non-working thing.
